# 学生情绪管理系统（管理员/辅导员端）测试使用说明

本文档用于给测试同学快速上手当前分支：`feat/student-emotion-web-only`。

---

## 1. 测试目标

- 验证登录、权限控制、账号管理、角色范围、档案查询、可视化、预警处置、审计日志、用户反馈是否可用。
- 验证“同一功能在不同角色下可见性差异”。
- 验证“功能之间的数据联动”是否符合预期（例如：预警处置是否写入审计、权限范围是否影响可视化和档案检索）。

---

## 2. 启动方式

### 2.1 推荐：一键启动双系统（管理员端 + 学生端）

在仓库根目录执行：

```bash
bash /root/emo_detect/scripts/dev-up.sh
```

启动后默认地址：

- 管理员端前端（本项目）：`http://127.0.0.1:5173`
- 管理员端后端（本项目）：`http://127.0.0.1:5001`
- 学生端前端（DigitalMan）：`http://127.0.0.1:5174`
- 学生端后端（DigitalMan）：`http://127.0.0.1:5000`

停止：

```bash
bash /root/emo_detect/scripts/dev-down.sh
```

### 2.2 单独启动管理员端

后端：

```bash
cd /root/emo_detect/student-emotion-web/backend
python app.py
```

前端：

```bash
cd /root/emo_detect/student-emotion-web
npm run dev
```

## 3. 环境与数据前提（非常重要）

### 3.1 数据库配置

管理员端后端读取 `.env` 中 MySQL 配置（至少需要）：

- `MYSQL_HOST`
- `MYSQL_PORT`
- `MYSQL_DATABASE`
- `MYSQL_USER`
- `MYSQL_PASSWORD`

### 3.2 演示账号是否自动生成

后端的演示账号/演示数据默认**不会自动灌入**。  
只有设置 `SEM_ENABLE_DEMO_SEED=true` 时才会写入默认账号和示例数据。

如果你们当前数据库里还没有账号，有两种方式：

1. 在启动后端前设置 `SEM_ENABLE_DEMO_SEED=true` 再启动；  
2. 或由已存在管理员在“账号管理”里手工创建。

## 5. 角色与权限矩阵

### 5.1 超级管理员 `SuperManager`

可见/可用：

- 主页总览
- 账号管理
- 角色/范围配置
- 预警阈值设置
- 安全审计日志
- 用户反馈（查看/删除）
- 修改密码

不可用：

- 意见反馈提交（超级管理员被禁止走该通道）

### 5.2 辅导员账号

可见/可用：

- 主页总览（含情绪趋势、分布、今日预警数、高危 Top3）
- 心理档案查询（仅管辖范围内学生）
- 预警与干预（仅可见/可处理权限内预警）
- 意见反馈提交
- 修改密码

不可用：

- 所有管理员页面（账号、角色范围、阈值、审计、用户反馈）

### 5.3 学院管理员（ADMIN 且带学院 scope）

可见/可用：

- 账号管理（受学院边界约束）
- 角色/范围配置（受学院边界约束）
- 主页总览、档案、预警（按管理员 scope 生效）

受限：

- 无权访问“预警阈值设置 / 安全审计日志 / 用户反馈”  
  （这些页面仅“全校级管理员/超级管理员”可访问）

## 6. 页面入口与功能互联关系

## 登录与会话

- 登录页：`/login`
- 登录后跳转：
  - `ADMIN` -> `/admin/accounts`
  - `COUNSELOR` -> `/counselor/archive`
- 冻结/停用账号会触发强制下线。

## 主页总览（`/dashboard`）

- 核心聚合页：展示趋势图、分布图、可见学生数、今日新增预警、高危 Top3。
- 与其他页面联动：
  - “今日新增预警”可跳到 `/counselor/alerts`
  - 分布图可点击扇区查看对应学生名单
  - 数据来源受当前登录人 scope 控制（角色/范围配置会影响这里）

## 心理档案（`/counselor/archive`）

- 检索学生（学号/姓名/关键词） -> 进入档案抽屉。
- 档案内容：基础信息（脱敏）、历史情绪曲线、时间轴、评估报告。
- 每次档案访问会写入审计日志（管理员可在审计页查看）。

## 预警与干预（`/counselor/alerts`）

- 查看预警列表（含等级、原因、状态、备注）。
- 可执行状态流转：
  - `NEW -> FOLLOWED`
  - `NEW/FOLLOWED -> CLEARED`
- 更新状态会写入审计日志。

## 账号管理（`/admin/accounts`）

- 查看账号状态、失败登录次数、锁定时间、最近登录。
- 管理动作：启用 / 冻结 / 停用 / 重置密码 / 新增账号。
- 冻结和停用会强制目标账号下线。

## 角色/范围配置（`/admin/role-scope`）

- 配置角色（管理员/辅导员）与数据范围（学院、年级、专业、班级）。
- 支持辅导员多辖区（`scopes`）配置。
- 配置完成后，直接影响：
  - 档案检索结果
  - 主页可视化统计
  - 预警可见范围

## 预警阈值设置（`/admin/thresholds`）

- 调整敏感度和各等级分数区间。
- 仅全校级管理员可访问。
- 修改会记录审计日志。

## 审计日志（`/admin/audit`）

- 查看关键操作链路：登录、冻结/停用、密码修改、角色配置、预警处置、反馈处理等。

## 反馈链路

- 教职工提交：`/feedback`
- 管理员查看/删除：`/admin/feedback`
- 超级管理员不可提交反馈，只能查看处理。

---

## 7. 推荐测试顺序（按链路）

1. **登录链路**：分别用管理员、辅导员登录，确认菜单差异。
2. **范围链路**：管理员在“角色/范围配置”修改某辅导员范围。
3. **档案链路**：辅导员重新登录后，验证档案检索结果变化（范围是否生效）。
4. **预警链路**：辅导员在“预警与干预”更新状态，回到管理员看审计日志是否记录。
5. **账号安全链路**：管理员冻结某账号，验证该账号是否被强制下线并禁止访问。
6. **阈值链路**：全校管理员调整阈值，确认可保存且有审计记录。
7. **反馈链路**：辅导员提交反馈 -> 管理员“用户反馈”页可见并可删除。

## 8. 常见问题排查

- **登录页有默认账号提示，但登录失败**  
  先确认是否开启 `SEM_ENABLE_DEMO_SEED=true` 或数据库里是否已有账号。

- **辅导员看不到某学生**  
  优先检查该学生是否在其 scope（学院/年级/专业/班级）内。

- **管理员看不到阈值/审计/用户反馈菜单**  
  该账号可能是“学院管理员（带学院 scope）”，不是全校级管理员。

- **预警有数据但趋势/分布观感不一致**  
  预警和可视化统计口径不同，属于预期（预警按规则聚合，可视化按可见学生聚合）。
